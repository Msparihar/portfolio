## Overview

Every interaction should feel alive. Windows don't just appear — they pop in smoothly. Books don't just open — they unfold like in anime. The portfolio OS should feel as polished as a real desktop OS, with thoughtful animations that reinforce each world's identity.

## Core Animation System

### Window Animations
- **Open:** Scale from 0.9 → 1.0 with opacity 0 → 1, slight bounce easing (200-300ms)
- **Close:** Scale 1.0 → 0.95, opacity 1 → 0 (150ms, ease-in)
- **Minimize:** Shrink toward taskbar icon position (like macOS genie effect, simplified)
- **Maximize:** Expand from current size to full screen (200ms, ease-out)
- **Focus:** Subtle lift (shadow intensifies) when bringing window to front
- **Drag start:** Very subtle scale up (1.0 → 1.01) to indicate "grabbed"

### Desktop Icon Animations
- **Double-click:** Icon bounces (scale 1.0 → 1.15 → 1.0) before window opens
- **Hover:** Slight lift + glow per world theme
- **Selection:** Highlight ring appears with fade-in

### Taskbar Animations
- **App icon bounce:** When a window first opens, its taskbar icon bounces once
- **Hover:** Slight scale up on taskbar icons
- **Active indicator:** Dot/line slides smoothly under active app

### Sidebar Animations
- **Expand/Collapse:** Smooth width transition (48px ↔ 200px) with content fade
- **Item hover:** Slide-right effect on nav items

### Context Menu
- **Open:** Fade in + slight Y translate (like dropdown appearing)
- **Close:** Quick fade out
- **Item hover:** Background color slides in from left

### Boot Sequence
- **Text lines:** Typewriter effect, each line appears character by character
- **Progress:** Smooth progress bar fill
- **Final reveal:** Desktop fades in from black/white depending on world

## Book / Codex Animation (KEY FEATURE)

### Book Icon
- A **book icon** appears in every world — in the taskbar, desktop, and sidebar
- Per-world styling: leather-bound (Elden Ring), picture book (Ghibli), illuminated manuscript (GoT)

### Book Opening Animation (anime-inspired)
- Click book icon → screen dims slightly (overlay)
- Book appears centered, closed
- **Pages unfold animation:** Like a real book opening — left cover swings open, pages fan out
- Think: anime book opening sequences (Howl's Moving Castle, Spirited Away opening credits)
- CSS 3D transforms: `perspective`, `rotateY`, `transform-origin: left`
- Cover opens from right to left (rotateY: 0 → -180deg)
- Pages inside fan slightly (staggered rotateY with delays)
- Content fades in on the right page after book is fully open

### Book Navigation
- **Page turn:** Click right edge → page flips right-to-left with 3D rotation
- **Page turn back:** Click left edge → page flips left-to-right
- **Chapter jump:** Pages rapidly flip (fast sequential animation) to target chapter
- **Close book:** Pages fold back, cover closes, book shrinks and flies back to icon position

### Technical
- Framer Motion or CSS keyframes + transforms
- `perspective: 1200px` on book container
- `transform-style: preserve-3d` for 3D page effects
- `backface-visibility: hidden` for double-sided pages
- Page content rendered on both sides (front = current page, back = next page)

## World-Specific Animation Themes

### Elden Ring
- **Easing:** Slightly heavy/dramatic — cubic-bezier(0.4, 0, 0.2, 1)
- **Window open:** Golden rune circle flash before window appears
- **Particles:** Golden embers drift upward on interactions
- **Boot:** Text burns in character by character (amber glow fade-in)
- **Transitions:** Dark fog wipe between states

### Ghibli
- **Easing:** Soft, bouncy — cubic-bezier(0.34, 1.56, 0.64, 1)
- **Window open:** Gentle pop with slight overshoot (like a flower blooming)
- **Particles:** Leaves/petals scatter on window open/close
- **Boot:** Seeds growing into text (organic reveal)
- **Transitions:** Soft watercolor dissolve

### GoT
- **Easing:** Regal, measured — cubic-bezier(0.25, 0.1, 0.25, 1)
- **Window open:** Varies by region:
  - North: Frost crystallizes into window frame
  - King's Landing: Gold shimmer reveal
  - Night's Watch: Stark/minimal fade
  - Dragonstone: Fire lick effect on edges
- **Boot:** Text appears as if written by quill (ink spread)
- **Transitions:** Banner unfurl / scroll unroll

## Codebase Audit Results (2026-04-09)

Haiku agent audited all 14 components. Key findings:

### Currently Existing Animations
- Window shadow transition (0.2s ease)
- Traffic light buttons (0.15s ease)
- Sidebar width toggle (200ms ease)
- Various hover transitions (0.1-0.15s on bg/color)
- Boot cursor blink (1s step-end infinite)
- Boot fade-out (0.8s ease)
- World switch canvas opacity (200ms)
- Several unused keyframes in globals.css (typing, fadeIn)

### High Priority Gaps (must-fix)
1. **Window open/close** — no animation at all, just instant render. Need scale(0.85→1) + fade (300ms)
2. **Window minimize** — just hides. Need shrink-to-taskbar animation (250ms)
3. **Desktop icon double-click** — no feedback. Need bounce pulse (300ms)
4. **Context menu** — instant appear. Need scale(0.95→1) + fade + cascade items (200ms)
5. **World switch** — only opacity fade. Need blur + color shift (400ms total)
6. **Gallery lightbox** — no entrance/exit. Need scale + fade (400ms)
7. **Mail success state** — no animation on send. Need checkmark scale-bounce (400ms)

### Quick Wins (high impact, easy to add)
- Icon hover: `scale(1.08)` on desktop + taskbar icons (150ms)
- Sidebar arrow rotation on toggle (180°, 200ms)
- Context menu scale-in (0.95→1, 200ms)
- Taskbar entrance: slide-up from bottom (300ms)
- FileManager rows: stagger fade-in on directory change (50ms intervals)

### CSS Keyframes Needed in globals.css
```css
@keyframes scaleIn { from { transform: scale(0.85); opacity: 0; } to { transform: scale(1); opacity: 1; } }
@keyframes fadeInScale { from { transform: scale(0.95); opacity: 0; } to { transform: scale(1); opacity: 1; } }
@keyframes slideUp { from { transform: translateY(20px); opacity: 0; } to { transform: translateY(0); opacity: 1; } }
@keyframes pulse { 0%, 100% { transform: scale(1); } 50% { transform: scale(1.4); } }
@keyframes bounce { 0% { transform: scale(1); } 40% { transform: scale(1.15); } 100% { transform: scale(1); } }
```

### Accessibility
- `prefers-reduced-motion` is configured in globals.css line 348 but NOT all components respect it
- ALL new animations must check this media query

## Technical Approach

### Option A: Framer Motion (recommended)
- Already in the React ecosystem
- Great for layout animations, exit animations, gesture support
- AnimatePresence for mount/unmount animations
- Good performance with hardware acceleration

### Option B: CSS-only
- Lighter weight, no dependency
- `@keyframes` + `transition` for most things
- Limited: can't easily do exit animations or layout animations
- Good for: hover states, simple transitions

### Recommended: Hybrid
- Framer Motion for: window open/close, book animation, boot sequence, world transitions
- CSS for: hover states, micro-interactions, simple transitions
- CSS custom properties for world-specific easing and colors
