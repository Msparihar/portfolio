# v0.5.2 — Visual Fidelity: Making Worlds Actually Look Different

**Status:** IN PROGRESS  
**Created:** 2026-04-09  
**Priority:** CRITICAL — without this, worlds are just color swaps

---

## Problem

Worlds currently change CSS variables (colors) but every component renders identically. The Pencil designs show fundamentally different visual treatments per world, but the code ignores all of it. Result: "normal color changing is not theme changing."

---

## What Must Change (Component-Level)

### 1. Desktop Icons — Currently emoji, need themed SVG/icons
**Current:** `{icon}` renders emoji like 📁 🖥️ ✉️
**Design shows:**
- Elden Ring: Gold rune-style icons on dark backgrounds, ornamental borders
- Ghibli: Soft nature-colored icons with rounded containers, leaf/flower motifs  
- GoT: House-colored icons with medieval styling

**Fix:** Create a world-aware icon system:
- `src/config/worldIcons.js` — maps `{ worldId: { appId: { icon: <SVGComponent>, label: 'string' } } }`
- DesktopIcon.jsx reads current world and picks the right icon set
- Fallback to emoji if no world-specific icon exists
- Icons should be SVG or CSS-based (not images) for color theming

**Files:** `DesktopIcon.jsx`, `DesktopIconGrid.jsx`, new `worldIcons.js`

### 2. Window Titlebar — Currently hardcoded macOS-style
**Current:** Hardcoded radial-gradient traffic lights, monospace font, basic layout
**Design shows:**
- Elden Ring: Ornamental border, Cinzel font title, gold gradient titlebar, chamfered edges
- Ghibli: Rounded 16px corners, Newsreader font, warm cream titlebar, leaf decoration
- GoT: Region-specific border color, serif font, stone/metal feel

**Fix:** Make titlebar style world-aware:
- Title font: use `var(--dt-heading-font)` instead of hardcoded `monospace`
- Traffic light colors: already done via CSS class overrides
- Titlebar background: already uses `var(--dt-titlebar-bg)` ✓
- Add world-specific title formatting (e.g., Elden Ring wraps title in `「 」`, Ghibli adds `✿`)
- Add world-specific border treatment (ornamental bottom border for Elden Ring)

**Files:** `WindowTitlebar.jsx`, `globals.css`

### 3. Window Chrome — No ornamental borders
**Current:** Simple box with shadow, same for all worlds
**Design shows:**
- Elden Ring: Thick gold border (`1px solid var(--dt-accent-border-strong)`), chamfered panel effect, inner glow
- Ghibli: Soft rounded corners (16px), subtle warm border, cloud shadow
- GoT: Region-colored border, stone texture feel

**Fix:** Add CSS variable for window border and world-specific styles:
- `--dt-window-border` CSS var per world
- `--dt-window-radius` CSS var per world (Elden Ring: 2px sharp, Ghibli: 16px round)
- World CSS classes for ornamental effects (grain, glow, etc.)

**Files:** `Window.jsx`, `globals.css`, `worlds.js` (add new vars)

### 4. Fonts Not Applied — Heading/body fonts are defined but unused
**Current:** `fonts: { heading: 'var(--font-cinzel)', body: 'var(--font-crimson-text)' }` in worlds.js, but NO component reads these
**Design shows:** Cinzel headings everywhere in Elden Ring, Newsreader in Ghibli

**Fix:** 
- Add `--dt-font-heading` and `--dt-font-body` to CSS vars per world
- Update components to use `font-family: var(--dt-font-heading)` for titles
- WindowTitlebar, app headers, sidebar labels, taskbar text should all use heading font
- Body text in apps should use `var(--dt-font-body)`
- Need to add Cinzel + Crimson Text fonts to layout.jsx (Elden Ring fonts)

**Files:** `worlds.js` (add font vars), `layout.jsx` (add fonts), `WindowTitlebar.jsx`, `Taskbar.jsx`, `Sidebar.jsx`, all app components

## New CSS Variables Needed (add to each world in worlds.js)

```javascript
// Font vars
'--dt-font-heading': 'Cinzel, serif',        // or Newsreader, IM Fell English
'--dt-font-body': 'Crimson Text, serif',      // or Geist, Geist
'--dt-font-mono': 'Geist Mono, monospace',

// Window vars  
'--dt-window-radius': '2px',                  // Elden Ring: sharp, Ghibli: 16px
'--dt-window-border': '1px solid rgba(201, 168, 76, 0.6)',

// Overlay vars
'--dt-overlay-vignette': 'radial-gradient(...)',
'--dt-overlay-grain-opacity': '0.03',
```

## New World Config Properties

```javascript
// Per world in worlds.js
brandText: 'ELDEN RING OS',           // taskbar branding
desktopWatermark: 'TARNISHED',         // large faded text on desktop (optional)
titleFormat: (title) => `「 ${title} 」`, // window title formatting (optional)
```

## Scalability

This pattern works for ANY future world:
- Add CSS vars to the world config → components adapt automatically
- Add world CSS class for structural differences → scoped overrides
- Add world config properties (brandText, watermark, titleFormat) → components read them
- No hardcoding world IDs in components — everything reads from config
