## Overview

When a first-time visitor lands on the application, instead of going directly to a desktop, they see a **world selector** — a visually stunning landing page where they choose which world to enter. This sets the tone, creates anticipation, and lets the user pick their experience.

## Landing Flow

1. Visitor arrives → animated intro (logo reveal, text fade-in)
2. World selector appears — 3+ world cards/portals arranged visually
3. Each world card shows: name, preview wallpaper, brief description, animated hover effect
4. User clicks a world → transition animation (portal opening, zoom-in, fade-through) → desktop loads with that world
5. Returning visitors can skip (go straight to last world) or re-choose

## World Cards

Each card is a "portal" into the world:
- **Elden Ring:** Dark portal with golden rune circle, ember particles, bonfire glow
- **Ghibli:** Soft watercolor doorway, leaves blowing, warm light inside
- **GoT:** Iron gate with house banners, torch flames, snow/fire effects per region

## Animations Required
- 3D card hover (perspective tilt, parallax depth)
- Portal opening transition (scale, blur, color shift)
- Particle effects per world (embers, leaves, snow)
- Text reveals (typewriter, fade, slide)
- Background ambient motion (subtle camera movement)

## Technical
- Separate route or overlay on top of main page
- Framer Motion for animations
- Three.js or CSS 3D transforms for portal effects
- localStorage remembers choice for returning visitors
- "Change World" option always available in taskbar/settings
