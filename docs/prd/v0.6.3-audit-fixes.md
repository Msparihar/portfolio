## Background

Haiku subagent audited `src/` for high-level bugs and inconsistencies. Five findings were investigated; two were false positives (BootOverlay already works correctly, whoamiName fallback is safely handled). The remaining four are patched here.

## Changes

### 1. Delete `src/components/desktop/ThemePicker.jsx`
Dead code. Removed from active use in commit `e9696dc` ("remove ThemePicker, redesign WorldPicker as visible pill dropdown") but the file was left behind. No imports anywhere.

### 2. Delete `src/config/worldIcons.js`
File is marked `// DEPRECATED — use worldContent.js instead` at the top. All functions duplicated in the newer typed `worldContent.ts`. Verified zero imports across `src/`.

### 3. `src/lib/email.js` — parameterize recipient
Both `sendVisitorAlert` and `sendContactMessage` hardcode `"manishsparihar2020@gmail.com"` as the recipient. Move to an `EMAIL_RECIPIENT` env var, falling back to the current address so existing deployments don't break.

### 4. `src/config/worlds.js` — Ghibli `brandText` consistency
Ghibli has `brandText: 'ghibli@portfolio'` which is a terminal-prompt style, while Elden Ring uses `'ELDEN RING OS'` and GoT uses `'WinterfellOS'` — both "OS" styled. Standardize Ghibli to `'GHIBLI OS'` to match the pattern.
