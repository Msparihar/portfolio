# PRD: Portfolio v0.5.0 — Worlds System + Dark Fantasy World

**Status:** Ready
**Date:** 2026-04-09

---

## Version Goal

Extend the existing theme architecture into a "Worlds" system — complete aesthetic transformations that swap fonts, wallpapers, CSS effects, and boot text alongside colors — and ship the first World: a dark fantasy Souls/Elden Ring aesthetic with parchment text, gold runes, grain textures, and chamfered panels.

---

## Background

v0.4.0 shipped the full desktop experience: sidebar, Trash, Gallery, live Mail, and Website Mode. The visual foundation (12 `--dt-*` color themes, `applyTheme()`, `BASE_THEME_VARS` delta pattern) is solid but limited to color swaps. Every theme looks structurally identical — same font, same window shape, same wallpaper style, same boot screen.

The v0.4.0 notes established a long-term "Worlds" vision: the portfolio as an explorable multiverse where each world is a completely different aesthetic experience wrapping the same content. v0.5.0 lays the architectural foundation and delivers the first non-trivial World.

**What a World adds beyond a simple color theme:**
- Custom fonts loaded via `next/font/google` (Cinzel + Crimson Text for Dark Fantasy)
- A CSS class applied to `.desktop-canvas` (e.g. `world-dark-fantasy`) for structural overrides: clip-path, grain textures, glow animations, dividers
- A `--dt-wallpaper-gradient` override that replaces the colored-blob style with something world-appropriate
- Custom boot screen lines (`bootLines` array in the World definition)
- A `worldClass` string — when non-null, signals components to render world-specific visual variants

**What must not break:**
- All 12 existing color themes continue working as "simple themes" — their `THEMES` array entries are unchanged
- The `applyTheme()` function signature is unchanged
- All v0.4.0 features: Sidebar, Trash, Gallery, Mail live send, Website Mode, ThemePicker
- Window dragging, resizing, minimize/maximize/close
- Boot overlay, terminal commands, context menu
- SEO pages (`/projects`, `/blog`, `/contact`)

## Features

### Feature 1: Worlds Architecture

**Description:** Introduce a `WORLDS` array in `src/config/worlds.js` (separate file — does not modify `themes.js`). Each World object contains: a color vars delta (using the same `--dt-*` token names as themes), fonts metadata, a wallpaper override, a CSS class name, and custom boot lines. A `WorldPicker` UI in the taskbar lets visitors switch Worlds. The active World is persisted in `localStorage`.

**World object shape:**

```js
{
  id: 'dark-fantasy',           // unique identifier
  name: 'Dark Fantasy',         // display name in picker
  swatch: '#c9a84c',            // color shown in picker swatch
  worldClass: 'world-dark-fantasy', // CSS class added to .desktop-canvas
  fonts: {
    heading: 'var(--font-cinzel)',      // CSS variable from next/font
    body: 'var(--font-crimson-text)',   // CSS variable from next/font
  },
  bootAccentColor: { text: '#c9a84c', ok: '#f9c043' },  // gold boot colors (also used for blink cursor)
  bootLines: [                  // replaces BOOT_LINES in BootOverlay when active
    'DARK FANTASY OS v1.0',
    'Copyright (C) unknown — The Age of Fire',
    '',
    'Communing with ancient runes.......OK',
    'Mounting the Erdtree...............OK',
    'Binding fog of war.................OK',
    'Summoning portfolio spirits........OK',
    'Igniting the bonfire...............OK',
    '',
    'Tarnished, your journey begins.',
    'May the grace guide thee...',
  ],
  vars: {
    // Full --dt-* delta (see Feature 2 for complete values)
  },
}
```

**A World with `worldClass: null` and `bootLines: null` is a simple theme — backwards compatible.** The existing 12 `THEMES` entries stay in `themes.js` and are unaffected. World definitions live exclusively in `worlds.js`.

**`applyWorld(canvasEl, worldId)` function** — exported from `worlds.js`. Works like `applyTheme()`:
1. Finds the World by id
2. Merges `BASE_THEME_VARS` + `world.vars` and sets CSS custom properties on `canvasEl` (same as `applyTheme`)
3. Removes any previously applied `world-*` class from `canvasEl.classList`, then adds `world.worldClass` if non-null
4. Does NOT handle fonts — font CSS variables are set globally in `layout.jsx`; the world class activates them via CSS

**`WORLD_STORAGE_KEY = 'portfolio_world'`** — separate from `THEME_STORAGE_KEY`. On mount, `Desktop.jsx` reads both keys and applies them in order: theme first, then world (world vars override theme vars).

**Interaction between themes and worlds:** A World is mutually exclusive with a simple theme — selecting a World deactivates the current theme, and selecting a simple theme deactivates the current World. `localStorage` stores either `portfolio_theme` (simple theme) or `portfolio_world` (world), never both at once. `ThemePicker` clears `portfolio_world` on theme select; `WorldPicker` clears `portfolio_theme` on world select.

**Smooth transition:** When switching worlds, a `200ms` `opacity: 0` fade is applied to `.desktop-canvas` before the class/variable swap, then `opacity: 1` is restored. This prevents the jarring visual pop of instant variable changes.

**Files:**
- `src/config/worlds.js` (new — World definitions + `applyWorld()` + constants)
- `src/components/desktop/WorldPicker.jsx` (new — World selection UI in taskbar)
- `src/components/desktop/Desktop.jsx` (modified — apply saved World on mount, listen for world changes)
- `src/components/desktop/Taskbar.jsx` (modified — render `<WorldPicker />` next to `<ThemePicker />`)
- `src/components/desktop/ThemePicker.jsx` (modified — clear `portfolio_world` on theme select)
- `src/components/desktop/BootOverlay.jsx` (modified — accept `bootLines` prop; fall back to built-in lines if null)

### Feature 4: World-Aware Components

**Description:** Three components get lightweight world awareness by reading the active `worldClass` from a new `useWorldStore` hook (or directly from localStorage on mount) and conditionally rendering world-specific attributes or elements. The rule: components do not contain world-specific logic — they expose hooks (`data-world-heading`, `data-world-accent`, `world-divider` class) that CSS targets. The exception is `BootOverlay`, which needs to swap its text content.

**4a. BootOverlay — world-specific boot text**

`BootOverlay.jsx` currently has a hardcoded `BOOT_LINES` array. Change it to accept an optional `bootLines` prop. If `bootLines` is provided and non-empty, use it; otherwise fall back to the built-in lines.

`DesktopPage.jsx` reads the active world from `localStorage` inside a `useState` initializer function. This avoids hydration mismatch because `useState(() => ...)` only runs on the client after hydration, and the boot overlay is already behind an `!booted` check that prevents flash.

```jsx
// In DesktopPage.jsx — read world boot config via useState initializer
const [worldBootConfig] = useState(() => {
  try {
    const worldId = localStorage.getItem(WORLD_STORAGE_KEY);
    if (!worldId) return null;
    const world = WORLDS.find(w => w.id === worldId);
    if (!world?.bootLines) return null;
    return {
      bootLines: world.bootLines,
      bootAccentColor: world.bootAccentColor ?? null,
    };
  } catch { return null; }
});

// In render:
<BootOverlay
  bootLines={worldBootConfig?.bootLines ?? null}
  bootAccentColor={worldBootConfig?.bootAccentColor ?? null}
  onBootComplete={() => setBooted(true)}
/>
```

Boot text color: the `BootOverlay` hardcodes `color: '#22c55e'` for the terminal text, `#4ade80` for "OK" markers, and `backgroundColor: '#22c55e'` for the blink cursor. In Dark Fantasy world the boot runs before CSS vars are active, so all three colors must be overridden inline based on the prop.

Add a `bootAccentColor` prop shaped as `{ text: string, ok: string }` (defaults to `{ text: '#22c55e', ok: '#4ade80' }` if null). For the Dark Fantasy world, `bootAccentColor` is `{ text: '#c9a84c', ok: '#f9c043' }` (gold). The blink cursor `backgroundColor` also uses `bootAccentColor.text`. Pass from `DesktopPage.jsx` based on active world.

The boot screen background stays `#000000` — this is world-agnostic and intentional (a cold black before the world loads).

**Changes:**
- `src/components/desktop/BootOverlay.jsx` — accept `bootLines` and `bootAccentColor` props
- `src/components/desktop/DesktopPage.jsx` — read active world on mount, pass boot props

`WindowTitlebar.jsx` currently renders only traffic-light buttons — the `title` prop is accepted but never displayed. Add a title text `<span>` centered in the titlebar after the traffic lights div. The span should have `data-world-heading=""` to activate Cinzel font in Dark Fantasy world.

Add this after the `</div>` that closes the traffic-lights container and before the closing `</div>` of the titlebar.

**Changes:**
- `src/components/desktop/WindowTitlebar.jsx` — add title `<span>` with `data-world-heading=""` after traffic lights

**4c. Terminal — `data-world-accent` on prompt**

In `src/components/TerminalContext.jsx` (the file that renders the desktop terminal via the `Terminal` export), add `data-world-accent=""` to the three prompt `$` character spans. These are:
1. History entries: `<span className="terminal-prompt text-green-500 mr-2 flex-shrink-0">$</span>` (approx line 538)
2. Boot sequence prompt: `<span className="text-green-500">{'>'}</span>` spans inside `BootSequence` component
3. Input prompt: `<span className="terminal-prompt text-green-500 mr-2">$</span>` (approx line 569)

**4d. `world-divider` class in app sections**

The `world-divider` class is a passive CSS hook. Components that have section headings should add `<div className="world-divider" />` after their primary heading. Suggested locations:

### Feature 5: WorldPicker UI

**Description:** A new `WorldPicker.jsx` component rendered in the taskbar directly to the left of the existing `ThemePicker`. Shows a globe/world icon button that opens a popover listing available Worlds with swatch, name, and a brief descriptor. Selecting a World applies it and persists to `localStorage`.

**Popover layout:**
```
┌──────────────────────────────────┐
│  Worlds                          │
│  ─────────────────────────────── │
│  ◉  Dark Fantasy    ✦ Medieval   │
│  ○  (more worlds coming v0.5.1+) │
└──────────────────────────────────┘
```

- Each world row: swatch circle (14px) + world name + short tag descriptor
- Active world has a checkmark `✓` or ring on the swatch
- A "None (use theme)" option at the top of the list — selecting it clears the active world and falls back to whatever theme is in `portfolio_theme`
- Popover background: `var(--dt-context-bg)`, border `var(--dt-accent-border-strong)`, `border-radius: 8px`, `padding: 10px 12px`, `z-index: 300`
- Popover closes on outside click or `Escape`
- Button icon: `✦` (a rune/star glyph that suits the Worlds concept), 13px, `color: var(--dt-text-muted)`. Active world: `color: var(--dt-accent)`.

**World switching transition:**
```js
const applyWorldWithTransition = (canvasEl, worldId) => {
  canvasEl.style.transition = 'opacity 200ms ease';
  canvasEl.style.opacity = '0';
  setTimeout(() => {
    applyWorld(canvasEl, worldId);
    canvasEl.style.opacity = '1';
  }, 200);
};
```

**User-facing impact:** First-time visitor sees no WorldPicker change — default world is null (the normal desktop experience). A visitor who clicks the `✦` icon in the taskbar can select "Dark Fantasy" and watch the entire OS transform: fonts swap, colors shift to gold/parchment, window panels get chamfered corners, boot text will be medieval next time they load the page.

## Out of Scope

- **Additional Worlds beyond Dark Fantasy** — Cyberpunk, Space, Harry Potter, Studio Ghibli are roadmap items for v0.5.1+. The architecture supports them; this version only ships one.
- **Display Options / System Preferences window** — deferred from v0.4.0, still deferred. WorldPicker is a lightweight taskbar control, not a full settings panel.
- **3D elements** — Three.js / React Three Fiber (v0.6.0)
- **Sub-worlds / project environments** — per-project explorable spaces (v0.7.0)
- **Screensaver** — deferred
- **Sound effects** — deferred
- **World-specific desktop icons** — all worlds use the same icon set; custom per-world iconography is v0.5.1+
- **World-specific terminal themes** — terminal prompt gets a rune glow in Dark Fantasy but full custom terminal color schemes (different ANSI colors, custom prompt format) are deferred
- **`IM Fell English` and `MedievalSharp` fonts** — research listed these but they add weight without sufficient payoff at v0.5.0 scope. `Cinzel` + `Crimson Text` cover 95% of the visual impact.
- **World-specific sidebar and context menu icons** — emoji icons remain unchanged across worlds in v0.5.0
- **Light/animated wallpaper for Dark Fantasy** — the wallpaper is a CSS gradient; a canvas-based animated fog or parallax background is v0.6.0
- **World persistence on boot screen flash prevention** — same one-frame flash as theme system; a blocking inline script fix is deferred
