# v0.6.2 — Ghibli Wallpaper Fix + next/image Migration

**Status:** READY
**Created:** 2026-04-15
**Depends on:** v0.6.0 (worlds system)

---

## Problem

The Ghibli world's wallpaper image is invisible in the running app. Users see only a pastel gradient instead of the authored Ghibli-themed wallpaper photo.

## Root cause

`src/config/worlds.js:166` sets Ghibli's `--dt-wallpaper-gradient` to an **opaque** linear gradient:

```js
'--dt-wallpaper-gradient': 'linear-gradient(160deg, #d4e8d0 0%, #e8f0e5 40%, #f0e8d8 75%, #faf6ed 100%)'
```

In `Desktop.jsx:213-230` the wallpaper image renders at `zIndex: 0` and the gradient layer sits on top at `zIndex: 1`. Because the Ghibli gradient uses solid hex colors with no alpha, it fully paints over the image.

All other worlds (Elden Ring, GoT regions) use semi-transparent `radial-gradient` stops (`rgba(..., 0.08-0.15)`) which tint the image without hiding it. Ghibli just didn't follow that pattern when v0.6.0 authored the world.

## Secondary issue — performance

The wallpaper renders via a plain `<img>` tag:
- No `next/image` optimization → no AVIF, no responsive srcset, no lazy loading
- Ghibli asset is 297 KB, Elden Ring 168 KB, GoT has 4 region wallpapers
- All served directly from origin (no CDN)

CDN migration is out of scope for this patch (tracked in IDEAS.md), but swapping `<img>` → `next/image` with `priority` gets us AVIF + srcset for free.

## Fix

### 1. Make Ghibli's gradient transparent
Rewrite to semi-transparent radial tints in the same spirit as Elden Ring, using Ghibli's green/cream palette. The image shows through; the gradient adds mood.

### 2. Migrate wallpaper to `next/image`
In `Desktop.jsx`, replace the `<img>` with `<Image fill priority>` from `next/image`. Keeps the same absolute-inset layout; Next handles format negotiation.

## Files touched
- `src/config/worlds.js` — rewrite Ghibli's `--dt-wallpaper-gradient` and `--dt-wallpaper-gradient-size`
- `src/components/desktop/Desktop.jsx` — swap `<img>` → `next/image`

## Acceptance
- Ghibli world shows the wallpaper photo with a soft green/cream mood gradient on top
- Other worlds (Elden Ring, GoT regions) still render identically
- Network tab shows AVIF served (or WebP fallback) when switching worlds
- No console errors from `next/image`

## Out of scope (deferred)
- Cloudflare CDN for `/images/*`
- Cloudinary migration
- Higher-resolution wallpapers (1920×1200+)
- world-identity-gaps fixes (taskbar icons, app titles, favicon, etc.) — possibly v0.6.4
